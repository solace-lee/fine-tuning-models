2. 硬件限制下的特别考虑

2.1 GTX1050 2G显存的技术约束

模型大小限制：

目标模型：参数量不超过1B的小型模型
推荐选择：TinyLlama-1.1B、Phi-2-2.7B、DistilBERT、MobileViT
避免选择：7B以上参数模型（显存不足）
量化策略：

优先使用INT8量化，减少75%内存占用
备选4-bit量化（QLoRA）进一步压缩
模型加载时自动量化，无需额外配置
优化技巧：

梯度累积替代大batch size
梯度检查点（checkpointing）节省激活内存
避免全参数微调，专注LoRA/QLoRA

2.2 备选方案与应急措施

CPU运行方案：

使用WebAssembly后端，利用32G内存优势
速度较慢但稳定可靠，适合学习实验

本地优化：

关闭其他显存占用程序
调整系统虚拟内存设置
优先使用Chrome/Edge（WebGPU支持较好）

5. 关键技术栈与工具

5.1 核心框架

TensorFlow.js（前端首选）

优势：成熟稳定、社区活跃、WebGPU支持
适用：自定义模型训练、计算机视觉
Transformers.js（LLM专精）

优势：Hugging Face生态、ONNX运行时、预训练模型丰富
适用：自然语言处理、文本生成
ONNX Runtime Web（生产部署）

优势：跨平台、高性能、多后端支持
适用：生产环境部署、性能关键应用

5.2 硬件加速方案

WebGPU（首选）

启用：Chrome flags开启"Unsafe WebGPU"
验证：navigator.gpu 检查支持
WebAssembly（兼容方案）

适用：旧设备、CPU运行、内存充足场景
工具：@tensorflow/tfjs-backend-wasm
WebNN（未来潜力）

优势：NPU专用加速、能效比高
现状：实验性特性，逐步支持

5.3 开发工具链

调试工具：

tf.util.enableDebugMode() 详细日志
tf.memory() 内存监控
Chrome DevTools Performance面板
可视化工具：

tfvis（TensorFlow.js可视化库）
自定义训练监控面板
性能优化：

Web Workers避免UI阻塞
